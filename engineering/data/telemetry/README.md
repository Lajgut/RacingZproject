# Телеметрия AIM — хранилище и соглашения

> Сырые данные с велокомпьютера AIM (AiM Solo / MXm / EVO4s и т.д.) и экспорты для анализа.

## ⚠️ Важно: вес пилотов для сравнения телеметрии

| Пилот | Вес | Целевое время (МРВ) |
|---|---|---|
| **Я (Кирилл)** | ⏳ мой вес | PB 1:40.6 |
| **Антон Фёдоров** | на 15 кг легче меня | 1:39 ← **референс, к которому стремимся** |

> Разница в 15 кг влияет на: торможение (кинетическая энергия), саг (→ настройка пружины не 1:1 копируется), ускорение (power-to-weight), нагрев/износ шин. **Сетап Антона нельзя копировать вслепую — компенсируем разницу в весе.**

## Структура
```
telemetry/
├── me/      # мои заезды
└── anton/   # референс Антона (1:39)
```

## Правила именования файлов
Формат: `YYYY-MM-DD_трек_условия_описание.<ext>`

Примеры:
- `2025-05-12_mrw_dry_quali.csv`
- `2025-05-12_mrw_dry_race-run.xlsx`
- `2024-08-03_mrw_dry_PB-1-40-6.csv` ← мой лучший круг

Где:
- **трек**: `mrw` (Москва Рейсинг Вэй), `igora`
- **условия**: `dry` / `wet`
- **описание**: `quali`, `race`, `long-run`, `PB-X-xx-x`

## Что кладём сюда
- ✅ Экспорт из AIM Race Studio: `.csv`, `.xlsx` (каналы: RPM, speed, throttle, brake, gear, GPS-позиция, секторные времена)
- ✅ Скриншоты графиков из Race Studio (trace-сравнения мой vs Антон)
- ✅ Логи секторных времён (S1/S2/S3)

## Что НЕ кладём
- ❌ Сырые бинарники AIM (`.stk`) — слишком тяжёлые и бесполезные без Race Studio. Храни их локально.

## Методика анализа (для инженера)
1. **Сравнение кругов:** мой vs Антон по каналам (speed trace, brake trace, throttle) → ищем, где теряем.
2. **Секторный анализ:** в каком секторе самая большая дельта до Антона.
3. **Corner-by-corner:** для проблемного сектора — точки торможения, min speed в апексе, прогрессия газа на выходе.
4. **Long-run consistency:** разброс圈-to-圈, выносливость.
